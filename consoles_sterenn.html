La programmation de jeux sur consoles

La programmation de jeux sur console est très vague, il en existe de toutes sortes. Chacune vise un public différent : les consoles Nintendo visent un public beaucoup plus jeune et les autres consoles sont destinées a un public un peu plus adulte. Il existe également deux types de console : les consoles portables conçues pour être facilement transportées et les consoles de salon. Récemment, la sortie de la Nintendo Switch a su combler ces deux demandes avec son socle connectable à la télévision et son mode portable

Les consoles rétros

Dans cette partie, nous allons nous consacrer au vieilles consoles qui caractérisent le début des jeux vidéos. Il est aujourd'hui possible à chacun d'entre nous de créer des jeux vidéos sur ces consoles.

La Super Nintendo (.S.NES)

Pour commencer nous allons parler de la Super Nintendo, un console dite de salon qui se joue donc sur grand écran, elle refait aujourd'hui surface avec la version mini(SNES). Cette console se code avec un langage dit de bas niveau qui est le ASM autrement appelé "L'Assembleur". Pour utiliser ce langage, il faut déjà avoir une petite connaissance de d'autres langages de programmation tels que le C par exemple. Pour programmer sur cette console on va manipuler les bits/octets directement. Pour programmer avec ce langage, il faut tout d'abord utiliser un assembleur comme wla-dx qui permet également de remplir tout seul l'en-tête du programme que l'on nomme ROM. Il se trouve dans deux adresses différentes.

L'Arduboy, La Gamebuino et La PocketCHIP

Ce sont trois consoles qui ressemblent à des consoles rétros portables mais qui ont la particularité de se coder avec un Arduino, donc avec le langage C et l'interface Arduino, ce qui permet assez aisément de coder des jeux avec, si l'on a une bonne maitrise du langage. Ces trois consoles peuvent être crées de toutes pièces par des utilisateurs lambda.

  • L'Arduboy ressemble physiquement à la première Gameboy la célèbre console portable.
  • La Gamebuino ressemble quant à elle a la dernière Gameboy Advance elle permet de gérer une sortie de sons, et de jouer en multijoueur avec un câble de liaison et le codage se fait à l'aide du protocole I2C.
  • Pour finir, la PocketCHIP possède un clavier QWERTY et des modules Wifi et Bluetooth.

Les consoles récentes

Les consoles portables Nintendo

Vous connaissez tous la console Nintendo DS et ses variantes. Aujourd'hui, il est possible de programmer sur la Nintendo DS avec le langage BASIC. Aujourd'hui, grâce à un jeu créé par Petit Computer, il nous est possible de créer des jeux en BASIC directement avec la console. C'est en effet ce langage qui nous permet de coder sur les consoles de la Nintendo DS à la Nintendo 3DS, c'est un langage qui ne possède aucune librairie externe comme les langages Python ou C peuvent posséder. C'est un langage qui nécessite un interprète, les anciens étant des interprètes par numéros de ligne, tandis que les modernes sont plus proches de ceux que nous connaissons avec l'arrivée de la programmation orientée objet et le stockage de variables dans la mémoire qui n'était pas possible avant pour le BASIC

La Nintendo Switch

Sur cette dernière console sortie par Nintendo il nous est également possible de créer des applications et donc des jeux vidéos. Le langage de programmation généralement utilisé sur console est le C++, il est également possible de coder en C# en utilisant des moteurs de jeux qui vont se charger de le transformer en code natif.
Le C++ est un langage qui permet d'utiliser la programmation orientée objet, procédurale et générique qui est nécessaire (notamment la programmation orientée objet) pour créer des jeux vidéos. Le C++ est en réalité la version suppérieure du Langage C (d'où son nom).
Le C# (sharp) est, quant à lui, un dérivé du C++, qui est très proche du Java : langage utilisé pour créer des applications Androïd. Le C# reprend la syntaxe du langage Java et y ajoute certaines notions (surcharge des opérateurs, indexeurs, délégués), mais il est plus utilisé pour la programmation web (pour des applications) que pour la création de jeux vidéos sur console.

La Xbox

C'est aujourd'hui grâce au Xbox Live Creators Program que nous pouvons créer des jeux vidéo et les publier sur le Xbox Store; le but étant de favoriser les petits développeurs et les studios indépendants. Sur la Xbox 360 par exemple les développeurs amateurs peuvent utiliser le langage C#, mais plus les plus grands développeurs c'est un langage différent que leur a fourni Microsoft.
Il existe trois types de jeux sur Consoles: les jeux indépendants, les jeux d'arcade et les jeux officiels qui sont les jeux fourni par Microsoft.
Pour les jeux indépendants, il faut utiliser XNA qui est une bibliothèque C# fournie par Microsoft. Pour les jeux d'arcade, il faut une licence qui donne accès à de meilleurs outils que XNA.
Dans les deux cas présentés ci-dessus, l'utilisation des librairies Xbox est payante et donne l'accès de la publication des jeux sur le store.
